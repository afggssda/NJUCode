## 5. 质量管理计划

- 代码规范：ruff/black + pre-commit
- 测试策略：单元测试 + 集成测试 + 回归测试
- 评审机制：PR 至少 1 名人类成员审查
- AI 产出策略：必须人工复核后合并
- 周期性检查：每周一次里程碑评审

------

## 6. 配置与变更管理

- 分支模型：`main / dev / feature/* / ai/*`
- 合并策略：Squash 合并，保留任务关联
- 变更记录：`CHANGELOG.md` + `docs/ai-ledger.md`
- 版本标记：中期版 `v0.5-milestone`

------

## 7. 风险管理

1. 进度风险：模块耦合导致延期
   - 对策：接口先行，按周联调
2. Windows兼容风险：路径/编码/命令差异
   - 对策：平台适配层 + Windows 专项回归
3. AI 质量风险：生成代码不稳定
   - 对策：限定职责 + 测试兜底 + 人工审查
4. 追踪风险：AI 来源不清晰
   - 对策：强制标识 + ledger + PR 模板字段

------

## 8. 沟通机制

- 每日站会（15分钟）
- 每周评审（进度、风险、质量）
- 缺陷看板与任务看板同步维护
- 智能体任务需有“任务单 + 审查记录”
