# 图形化界面改动记录 2026.3.30 Zhou Jingyu

## 1. 改动目标
轮改动聚焦于 Textual 图形化界面的布局可用性与交互体验，核心目标如下：

1. 调整三栏布局，使代码工作区居中、聊天区在右侧。
2. 支持鼠标拖拽分栏宽度，并处理隐藏/恢复的交互细节。
3. 优化底部信息展示，补充运行上下文（分支、Python、模型等）。
4. 将 Chat 显示/隐藏从按钮方案调整为底部快捷键方案（类似 `^q Quit`）。

## 2. 主要改动内容

### 2.1 布局结构调整
1. 将 `Chat` 面板移动到右侧栏。
2. 将 `Code / Tools / Model` 标签页移动到中间栏。
3. 文件树点击文件后，自动切换到中间栏 `Code` 标签页。

### 2.2 分栏拖拽与隐藏/恢复
1. 左右分割线支持鼠标拖拽调整宽度。
2. 拖拽结束后保存当前分栏比例，下次启动自动恢复。
3. 隐藏策略采用阈值+滞回：
- 左栏显示阈值 `0.12`，隐藏阈值 `0.10`
- 右栏显示阈值 `0.14`，隐藏阈值 `0.12`
4. 右侧 Chat 隐藏后可通过快捷键快速恢复（见 2.4）。
5. 修复了“拖左侧影响右侧宽度联动变化”的问题，左右拖拽逻辑已解耦。

### 2.3 状态栏增强
1. 新增底部状态信息行，展示：
- `Branch`
- `Python` 版本
- `Mirror`
- `Model`
2. 模型配置保存后，状态栏会同步刷新显示。

### 2.4 Chat 显示/隐藏交互（最终方案）
1. 不再使用页面按钮进行 `Show/Hide Chat`。
2. 改为底部快捷键提示风格（同 Footer）：
- `Ctrl+H` -> `Show/Hide Chat`
3. 显示 Chat 时按默认宽度恢复：
- `default_chat_ratio = 0.34`

### 2.5 聊天气泡显示优化
1. 用户/助手/系统提示/系统错误消息分层渲染。
. 时间戳从正文中拆分为独立行显示，减轻正文视觉干扰。

## 3. 样式与主题处理
1. 本轮最终保留原始配色体系（未切换到新主题）。
2. 修复了 Textual CSS 不兼容写法：
- 移除不支持的 `:active` 伪类
- 移除不支持的 `color: inherit`
3. 分割线样式调整为细边界线（非粗按钮式）。

## 4. 关键快捷键（当前）
1. `Ctrl+N`：新建会话
2. `Ctrl+H`：显示/隐藏 Chat
3. `Ctrl+C`：中断模型流式输出
4. `Ctrl+Q`：退出

## 5. 涉及文件
1. `frontend/app.py`
2. `frontend/app.tcss`
3. `frontend/state.py`
4. `frontend/ui/widgets/splitter.py`
5. `frontend/ui/widgets/chat_panel.py`

## 6. 验证结论
1. 上述改动后，样式解析错误已清理。
2. 在 `conda` 环境 `nju_code` 下执行 `python main.py` 可启动运行。
3. Chat 栏可通过 `Ctrl+H` 在隐藏/显示间切换，显示时恢复默认宽度。
