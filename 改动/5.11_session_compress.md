# 2026-05-11 Session 与上下文压缩功能改进记录

## 一、概述

本次改动在已有 Session 管理与上下文压缩基础上，对各模块进行了系统性的深度改进。核心目标是提升压缩质量、增强稳定性、改进持久化能力，以及优化用户界面交互体验。

核心变更方向：
1. **双语 Token 估算** — 新增 CJK/ASCII 分离计算，显著提升中文内容的 token 估算精度
2. **增量压缩与历史追踪** — 支持摘要叠加，保留完整压缩历史统计
3. **摘要质量保障** — 引入关键词校验与多次重试机制，降低摘要丢失信息的风险
4. **设置持久化增强** — 自动备份 settings.json，导出文件生命周期管理
5. **状态层能力扩展** — 会话克隆、统计查询、批量重算、自动标题等新方法
6. **面板 UI 改进** — 移除手动压缩按钮、新增会话搜索过滤、丰富统计显示
7. **App 层集成改进** — 自动标题生成、压缩通知携带 token 数量、启动时重算估算值

## 三、各模块改进详解

### 3.1 `frontend/services/context_compressor.py`

**Token 估算改进 — 双语分离计算**

原有估算方式以字符数整体除以 3，对中文内容误差较大（中文字符编码后在 GPT tokenizer 中通常约 1 token/字）。改进后：

- `_count_cjk_chars()` 通过 Unicode 范围检测 CJK、平假名、片假名、谚文字符
- CJK 字符按 1.5 字符/token、ASCII 字符按 4 字符/token 分别估算
- `estimate_message_tokens()` 每条消息额外加 4 token overhead（role + 边界）
- `estimate_text_tokens()` 用于摘要文本估算（不加 overhead）
- 新增 `get_token_usage_ratio()` 返回当前用量与阈值的比值浮点数

**增量压缩**

`compress()` 方法新增 `existing_summary` 和 `session_title` 参数：

- 若 `existing_summary` 不为空，`_build_summary_prompt()` 将其作为历史背景附加到压缩提示中，使新摘要"叠加"而非"覆盖"旧摘要
- `_compute_adaptive_keep_recent()` 动态计算保留消息数量，目标是让保留部分约占 token 阈值的 50%

**摘要质量保障**

- `_validate_summary()` 检查摘要是否包含 `【用户意图】` 或 `【关键结论】` 标记之一，且最短 30 字符
- `generate_summary()` 最多重试 `max_summary_retries`（默认 2）次；若最终仍不通过验证，调用 `_build_fallback_summary()` 生成结构化降级摘要
- `generate_summary()` 返回 `(str, bool)` 二元组，第二项表示是否使用了降级摘要

**压缩历史统计**

新增 `CompressionRecord` dataclass 记录每次压缩的元数据：

```python
@dataclass
class CompressionRecord:
    compressed_at: datetime
    messages_removed: int
    token_before: int
    token_after: int
    tokens_saved: int
    summary_length: int
    used_fallback: bool
    session_title: str
```

配套方法：`get_compression_history()`、`get_total_tokens_saved()`、`get_compression_count()`、`format_compression_stats()`

### 3.2 `frontend/services/settings_store.py`

**备份机制**

`save()` 在写入新数据前先将当前 `settings.json` 备份为 `settings.backup.json`，配套 `restore_from_backup()` 和 `has_backup()` 方法。

**导出文件生命周期管理**

- `list_export_files()` 读取 `exports/` 目录，返回 `List[Tuple[Path, datetime, str]]`（路径、修改时间、会话标题），已按修改时间倒序排列
- `_read_export_title()` 快速读取导出文件标题，不加载消息
- `read_export_header()` 返回除 `messages` 字段外的导出文件头部字典
- `cleanup_old_exports(keep_count=20)` 保留最新 N 个，删除超出部分，返回删除数量
- `get_exports_dir_size_bytes()` 统计导出目录总占用字节数

### 3.4 `frontend/state.py`

**新增方法**

| 方法 | 说明 |
|------|------|
| `get_session_by_id(session_id)` | 替代 for 循环查找，提升可读性 |
| `get_session_stats(session_id)` | 返回详细统计字典，含消息数、用户/助手分类、token 估算、压缩次数、摘要长度、时间等 |
| `recalculate_token_estimates(compressor)` | 用双语估算器重新计算所有会话的 `token_estimate` |
| `auto_title_session(session_id, max_chars=30)` | 若标题仍为默认值，从首条用户消息生成标题；跳过 `/` 命令消息 |
| `prune_empty_sessions()` | 删除无消息的空会话（保留当前活跃会话），返回删除数量 |
| `clone_session(session_id)` | 深拷贝消息与摘要到新会话，标题加"（副本）"后缀 |

**已有方法改进**

- `compress_session()` 现为增量模式：传入 `existing_summary` 和 `session_title`，并在成功后更新 `compression_count` 和 `last_compressed_token_count`
- `auto_compress_if_needed()` 返回 `bool` 表示是否执行了压缩
- `build_context_messages()` 过滤 `summary`/`compressed` 等仅供 UI 展示的角色，避免将摘要气泡重复发送给模型
- `import_session()` 增加重复检测（比对标题 + 消息数 + 摘要），防止重复导入
- `load()` / `save()` 兼容新字段 `compression_count`、`last_compressed_token_count`
- `export_session()` 导出时包含新字段

### 3.5 `frontend/ui/widgets/session_panel.py`

**移除手动压缩入口**

删除 `SessionCompressRequested` 事件类和 `compress_session_btn` 按钮及其处理方法，由自动压缩机制完全替代。

**新增会话搜索过滤**

顶部新增 `session_search_input` 输入框，实时按标题过滤会话列表（大小写不敏感）。

**列表项丰富显示**

- `_build_session_label()` 对已压缩会话显示 `[C×N]` 标记（N 为压缩次数）
- `_format_relative_time()` 计算相对时间："刚刚"、"N 分钟前"、"N 小时前"、"N 天前"
- `_format_token_label()` 显示 `~1234 tokens (41%)`，超 80% 显示"⚠ 即将压缩"，超 100% 显示"⚠ 已超限"
- `refresh_sessions()` 接收 `token_threshold` 参数，用于百分比计算

**统计信息区域**

- `session_stats_label` 显示：`"{N} 条消息（用户 {N} 条） | 已压缩 N 次 | 上次: X 分钟前"`
- `session_summary_label` 显示摘要第一行（最多 60 字符）

### 3.6 `frontend/ui/widgets/chat_panel.py`

**压缩分隔线元数据**

`_build_compressed_divider(session)` 接受可选的 `ChatSession` 参数，生成如：

```
── 以上内容已压缩 · 共压缩 2 次 · 最近: 2026-05-11 14:32 · 节省 ~800 tokens ──
```

**新增辅助方法**

- `_format_message_time(dt)` — 同天显示 HH:MM，跨天显示 MM-DD HH:MM
- `_format_compression_info(session)` — 格式化分隔线附加信息文本

**气泡类型扩展**

`_build_bubble()` 新增 `error` 和 `system` 角色的显式 CSS 类处理，确保系统消息、错误消息与普通助手消息视觉上有所区分。

**渲染触发改进**

`render_messages()` 的会话签名现在包含 `compression_count` 字段，确保每次压缩后触发重绘，避免分隔线不刷新。

### 3.7 `frontend/app.py`

**移除手动压缩集成**

- 从导入列表中删除 `SessionCompressRequested`
- 原 `on_session_compress_requested()` 保留为空处理器（防止旧版事件残留报错），不再执行实际压缩

**自动标题生成**

新增 `_maybe_auto_title_session(session_id)` 方法：

- 在 `on_message_submitted()` 中用户发送第一条消息后立即调用
- 仅对标题仍为默认值（`"New Chat"` 或 `"Chat N"`）的会话生效
- 标题从首条非命令用户消息中截取，最长 30 字符
- 生成后保存状态并刷新会话列表

**启动时重算 Token 估算**

`on_mount()` 中 `state.load()` 之后调用 `recalculate_token_estimates(self.compressor)`，确保所有会话使用新双语估算器的值而非旧版估算残留。

**压缩通知改进**

`_finish_stream()` 中自动压缩发生后弹出通知，包含压缩前 token 数量：

```
上下文过长，已自动压缩（此前 ~4200 tokens）。
```

**`refresh_sessions` 传参改进**

`refresh_ui()` 和 `_refresh_active_chat_view()` 中的 `session_panel.refresh_sessions()` 调用均传入 `token_threshold=self.compressor.token_threshold`，使 Session 面板能正确计算 token 使用百分比。

**导出文件管理**

`on_session_export_requested()` 导出后调用 `cleanup_old_exports(keep_count=20)` 清理过旧文件，并在通知中说明清理数量。`on_session_import_requested()` 改用 `settings_store.list_export_files()` 获取文件列表（已按修改时间排序），减少冗余文件系统调用。

## 四、不兼容变更说明

| 变更点 | 影响 |

| `generate_summary()` 返回值改为元组 | 原返回 `str`，现返回 `(str, bool)`；调用方需解包 |
