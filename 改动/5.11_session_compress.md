# 2026-05-11 Session 与上下文压缩 改动记录


## 一、概述

本次改动实现了 Session 管理增强与上下文压缩功能，解决长会话超出模型 token 限制的问题。

核心变更：
1. **数据模型扩展** — `ChatSession` / `ChatMessage` 新增压缩相关字段
2. **上下文压缩核心** — 新建 `context_compressor.py`，token 估算 + 摘要生成 + 消息裁剪
3. **设置存储扩展** — 支持单会话 JSON 导出/导入
4. **状态管理扩展** — 8 个新方法，`load/save` 兼容新字段
5. **Session 面板 UI** — 新增 Compress/Export/Import 按钮与 Token 估算显示
6. **Chat 面板 UI** — 压缩分隔线、摘要气泡、中断恢复预填
7. **App 层集成** — 自动压缩、事件处理、中断上下文保存

## 三、新增模块详解

### 3.1 `frontend/services/context_compressor.py`

```python
@dataclass
class CompressionResult:
    summary: str          # 模型生成的历史摘要
    kept_messages: List[ChatMessage]  # 保留的近期消息
    removed_count: int    # 移除的消息数量
    token_before: int     # 压缩前 token 估算
    token_after: int      # 压缩后 token 估算

class ContextCompressor:
    def estimate_tokens(messages)   # 字符数 ÷ 3 近似 token
    def needs_compression(messages) # 是否超过阈值（默认 3000 token）
    def compress(messages)          # 执行压缩，返回 CompressionResult
```

- 无 API Key 时降级为拼接前几条内容，不崩溃
- 调用模型生成摘要复用现有 `OpenAICompatibleClient.chat()`

## 四、state.py 新增方法

| 方法 | 说明 |
|------|------|
| `get_token_estimate(session_id)` | 返回指定会话 token 估算值 |
| `compress_session(session_id, compressor)` | 执行压缩，写回摘要与压缩时间 |
| `auto_compress_if_needed(compressor)` | 当前会话超阈值则自动压缩 |
| `export_session(session_id, path)` | 导出会话为 JSON 文件 |
| `import_session(path)` | 从 JSON 文件恢复会话并追加 |
| `mark_interrupted(session_id, content)` | 标记中断恢复状态 |
| `clear_interrupted(session_id)` | 清除中断标记 |
| `build_context_messages(session_id)` | 构建含摘要的模型消息列表 |

## 五、UI 变更说明

### Session 面板

- 新增 `[C]` 标记：已压缩的会话在列表中显示 `● Chat 1 [C]`
- 新增 `Compress` 按钮：手动触发当前会话压缩
- 新增 `Export` 按钮：导出至 `.nju_code/exports/session_XXXXXXXX.json`
- 新增 `Import` 按钮：从 exports 目录导入最新一个会话文件
- Token 估算标签：实时显示 `~1234 tokens`
- 摘要预览标签：显示摘要前 80 字符

### Chat 面板

- 有压缩摘要时，消息列表开头显示 `── 以上内容已压缩 ──` 分隔线
- 摘要内容渲染为独立的摘要气泡（`bubble-summary` 样式类）
- 切换到中断会话时，输入框自动预填上次未发送内容

## 六、app.py 集成逻辑

- `__init__` 中创建 `self.compressor = ContextCompressor(self.client, self.state.model_config)`
- `on_message_submitted` 改用 `state.build_context_messages()` 构建发送给模型的历史（自动包含摘要前缀）
- `_finish_stream` 结束后调用 `auto_compress_if_needed()`，压缩时弹出通知
- `_stream_assistant_reply` 中断时调用 `mark_interrupted()` 保存输入内容
- 新增 `on_session_compress_requested`、`on_session_export_requested`、`on_session_import_requested` 三个事件处理方法
