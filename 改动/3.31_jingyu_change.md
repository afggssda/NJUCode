# 3.31 Change Log

本文档汇总 2026-03-31 在 `NJUCode` 项目中的主要改动，覆盖界面交互、模型调用链路、检索分析能力与文档整理。

## 1. 架构与目录调整
- 完成前端分层：`frontend/ui`、`frontend/services`、`frontend/models.py`、`frontend/state.py`。
- 保持 `main.py` 作为统一入口。
- 配置持久化目录固定为 `.nju_code/settings.json`。

## 2. TUI 界面与交互改进
- 三栏布局稳定：Explorer / Code(Tools,Model) / Chat。
- 支持分割条拖拽调整宽度，支持显示/隐藏聊天栏。
- 快捷键完善：
  - `Ctrl+N` 新建会话
  - `Ctrl+H` 显示/隐藏聊天栏
  - `Ctrl+C` 中断模型流式输出
  - `Ctrl+Q` 退出应用
- 状态栏增强：展示 Git 分支、Python 版本、镜像、模型。
- 聊天气泡样式优化（用户/助手/系统提示/错误区分）。
- Chat 区滚动行为优化，支持更稳定的历史消息浏览。

## 3. 文件管理与编辑能力
- 文件树支持打开工作区（Open Folder）。
- 新增文件/目录（`N`）与删除（`Delete`）能力。
- 文件树定时刷新并尽量恢复展开节点和光标位置。
- 代码查看器支持：
  - 打开文件与语法高亮
  - `Ctrl+S` / Save 保存
  - Reload 重载
  - Ask in Chat 快速把文件路径加入输入框

## 4. 会话与状态管理
- 多会话创建/切换/重命名/删除完整可用。
- 会话、工具开关、模型配置、分栏宽度均可持久化。
- 支持从 `.env` 覆盖模型配置关键字段。

## 5. 模型接入与上下文链路
- 完成 OpenAI 兼容流式调用封装：`frontend/services/openai_client.py`。
- 修复请求参数不一致问题：
  - `messages` 历史消息传递统一
  - 新增 `file_contexts` 字段
- 打通 `@文件路径` 上下文注入：
  - 发送前解析消息中的 `@xxx`
  - 读取物理文件文本
  - 作为系统上下文注入模型请求
- 支持模型配置中的 `model_file` 作为额外系统上下文。

## 6. Tools 面板增强（检索工作台）
- 在 Tools 面板新增 `Analysis Workbench`：
  - 参数输入：`query`、`path`、`depth`、`top_k`
  - 按钮：`Help`、`Scan`、`Search`、`Symbol`、`Summary`、`Deps`、`Recall`、`Impact`
- 按钮触发与聊天输入 `/命令` 共用同一分析执行链路。

## 7. 检索与多文件分析能力（WBS-3 核心）
新增 `frontend/services/code_analysis.py`，实现以下本地分析能力：
- 项目扫描与索引：递归扫描，过滤 `.git`、`venv`、`.venv`、`node_modules`、`__pycache__`。
- 文本检索：关键词/正则/大小写控制，返回路径+行号+片段。
- 符号检索：识别 Python `class/def/async def/import`，按符号名定位定义。
- 文件摘要：输出主要类函数、依赖、入口函数、作用说明。
- 多文件关联分析：基于 import 构建依赖图，支持 1~2 层邻接。
- 任务召回：自然语言 Top-K 文件召回（关键词+符号+路径权重）。
- 影响面分析：给定文件/符号输出影响列表、风险等级、建议阅读顺序。

## 8. 输出体验优化
- 分析结果默认输出为人类可读文本。
- 不再在聊天窗口附带 JSON 块。
- `search` 输出可读性增强：
  - 按文件分组
  - 控制每文件展示条数
  - 片段自动截断
  - 给出“未展示结果数”提示

## 9. 文档与记录
- 新增并整理 `进度/进度.md`、`进度/图形化界面改动记录.md`、`进度/idea.md`。
- README 更新为“当前真实能力 + 下一步规划”版本（见 `README.md`）。

## 10. 当前可直接使用的命令
可在聊天输入区，或 Tools 工作台按钮触发：
- `/help`
- `/scan`
- `/search <keyword> [--regex] [--case]`
- `/symbol <name>`
- `/summary <relative_path>`
- `/deps <relative_path> [--depth 1|2]`
- `/recall <requirement text> [--top 5..30]`
- `/impact <symbol_or_relative_path> [--depth 1|2]`
