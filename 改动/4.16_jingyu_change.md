# 4.16 Change Log

本文档记录 2026-04-16 在 `NJUCode` 项目中的主要改动，重点覆盖聊天自动上下文、文件管理安全性、界面交互修复与文案统一。

## 1. 聊天自动上下文增强
- 普通聊天消息新增“自动上下文收集”能力，不再只依赖手动 `@文件路径`。
- 支持从自然语言问题中自动识别：
  - 文件名 / 相对路径
  - Python 函数名 / 类名
- 命中后会自动执行本地检索，并将以下内容拼接进模型请求：
  - 对应文件全文
  - 文件摘要
  - 符号定义附近的局部上下文
- 自动上下文链路接入 `frontend/app.py`，与原有聊天发送流程复用同一请求结构。

## 2. 自动检索排序优化
- 针对“同名符号在多个文件中同时命中”的情况，新增本地排序与裁剪逻辑。
- 评分时综合考虑：
  - 用户消息中是否显式提到相关文件
  - 路径词与问题关键词的重合度
  - 用户问的是类还是函数
  - 定义位置与路径长度
- 自动注入上下文时，默认只保留最相关的 1~2 个命中，减少噪音上下文。

## 3. Code Viewer 行为修复
- 修复首次点击文件时，`NJU` 空态未正确消失的问题。
- 原问题表现为：第一次点击 `main.py` 等文件后代码不显示，必须切换到别的文件再回来。
- 现已改为：只要文件成功加载，就统一退出空态并显示编辑器。

## 4. 文件树与删除安全性增强
- 文件树中的新建提示文案改为更明确的成功提示。
- 删除文件/目录不再直接物理删除，而是先移动到 `.nju_code/trash`。
- 新增 `Undo Delete` 按钮与 `Ctrl+Z` 快捷键。
- 从“只支持最近一次删除撤销”升级为“支持连续多次删除撤销”。
- 撤销逻辑改为基于删除历史栈，按后删先恢复的顺序依次回退。

## 5. 分栏交互修复
- 修复左侧 Explorer 面板被拖拽隐藏后，向右拖拽分割条无法重新显示的问题。
- 同步补强右侧 Chat 面板隐藏后的恢复逻辑。
- 当前逻辑为：隐藏状态下，只要用户反向拖动对应分割条，就立即恢复显示。

## 6. Explorer / Session UI 文案与交互整理
- 删除启动时顶部弹出的两条提示通知，减少打开应用时的干扰。
- 移除左侧 Explorer 面板内部重复出现的 `Explorer` 标题，仅保留按钮区。
- 文件删除确认弹窗改为英文：
  - `Delete this item?`
  - `Cancel / Delete`
- 会话删除确认弹窗改为英文：
  - `Delete chat`
  - `Are you sure you want to delete this chat?`
  - `Cancel / Delete`
- 文件树相关通知统一改为英文，规避当前终端环境下中文提示首字缺失/乱码问题。

## 7. 当前结果
- 项目已具备：
  - 聊天 + 自动文件/符号上下文检索
  - 本地代码分析工作台
  - 文件安全删除与多次撤销
  - 更稳定的代码查看与分栏交互
- 下一阶段重点仍然是 WBS-4：Patch / diff / 回滚链路。
