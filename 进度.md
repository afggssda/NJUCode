🚀 您接下来最迫切需要推进的工作：WBS-3 (上下文分析与检索)
既然您已经有了一个“可以对话的框”和一个“可以看代码的资源管理器”，那么让 AI 能够“看懂您项目里的代码” 是让它成为“编程助手（Code Agent）”的第一步。目前右侧聊天的 AI 还处于“盲人”状态。

具体要做的事情 (WBS-3 的 MVP 实现)：
添加 "引用上下文" 的交互入口 @ (Add to Context)
在右侧对话框的输入处，或者在左侧资源管理的右键菜单（快捷键绑定），支持把某些特定文件 包含为上下文（Context）。例如输入 @main.py 解释一下这个文件。
实现本地文件读取转 Prompt
在发给 API 之前，劫持并解析带有 @（或已附在状态里）的文件列表，读取其绝对路径下的实际物理文本。
拼接成特定的 System Message (例如：Here is the context of file {main.py}:\n \``...\n````) 发给大模型。
实现全局模糊搜索能力 (选做能力提升)
当无法精确定位文件时，通过正则或关键词去扫描并返回整个目录树的代码段给大模型。
一旦完成 WBS-3，您的程序就从“普通的聊天壳”变为了“真正好用的代码智能问答工具”，随后就可以顺理成章地推进大模型主动帮您写代码的阶段 (WBS-4 Patch 系统)。
