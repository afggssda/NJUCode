# 需求文档（SRS）

## 1. 项目概述

### 1.1 项目名称

CLI Code Agent（类 Claude Code）——基于 Windows 的工程化命令行智能编程代理

### 1.2 项目背景

本项目为软件工程综合实验大作业，由 6 名人类成员 + 1 名智能体成员协作完成，目标是在命令行环境中实现具备代码理解、工程级修改、技能扩展、会话管理和成本监控能力的 Code Agent 系统。

### 1.3 项目目标

1. 构建可在 Windows + VS Code 环境稳定运行的 CLI Code Agent。
2. 实现“代码理解与解释、多文件上下文分析、工程级代码修改、Skills 扩展”四大核心能力。
3. 将大语言模型明确设计为“智能体团队成员”，承担可追踪、可审计的明确任务。
4. 最终达到课程项目规模和工程化要求（团队协作、过程管理、可验收）。

------

## 2. 项目范围

### 2.1 In Scope（纳入范围）

- CLI 交互与命令系统
- 多轮会话管理与上下文压缩
- 代码检索、结构分析、跨文件影响分析
- Patch 生成、预览、应用、回滚
- Skills 插件机制
- 模型路由与 token/成本监控
- AI 代码来源标识与过程追踪

### 2.2 Out of Scope（不纳入本期）

- 完整 IDE 图形界面（GUI）
- 分布式多节点部署
- 大规模企业级权限系统

## 4. 功能需求

### FR-01 命令行基础能力

- 支持命令：`init / chat / explain / analyze / edit / patch / session / skill / model / cost / doctor`
- 支持 `--help`、参数校验、错误提示

### FR-02 项目上下文读取

- 读取目录树、关键配置文件、规则文件（如 `AGENT.md`）
- 支持按路径/模块范围分析

### FR-03 代码理解与解释

- 对类、函数、模块进行解释
- 输出调用关系、职责说明、风险点

### FR-04 多文件上下文分析

- 根据需求自动检索相关文件
- 输出影响文件清单和修改建议

### FR-05 工程级代码修改

- 生成可审阅的 diff/patch
- 支持 dry-run、用户确认、失败回滚

### FR-06 会话与上下文管理

- 会话创建、恢复、切换
- 长上下文压缩与摘要保留
- 中断恢复机制

### FR-07 Skills 插件机制

- 插件安装/卸载/列表/调用
- manifest 定义输入输出与权限
- 插件执行审计日志

### FR-08 模型与成本管理

- 模型切换与任务路由
- token 使用统计与费用估算
- 预算阈值告警

### FR-09 Git/GitHub 开发辅助

- 生成 commit message、变更总结、PR 描述草案
- 分析改动影响与测试建议

### FR-10 AI 产出标识与追踪（课程强制要求）

- 对 AI 生成代码做自动标识
- 记录 AI 任务来源、输入、输出、审查人、合并记录
- 支持中期/终期展示追踪链路

------

## 5. 非功能需求

### NFR-01 平台兼容

- Windows 10/11 可用，VS Code 终端稳定运行

### NFR-02 性能

- 常规命令快速响应（不含模型推理）
- 长任务支持流式输出

### NFR-03 可维护性

- 模块化结构，统一编码规范
- 单元测试与集成测试

### NFR-04 安全性

- 高风险操作二次确认
- 敏感信息脱敏
- 插件最小权限原则

### NFR-05 可追踪性（课程重点）

- 人工/AI 贡献可区分
- 过程日志可追溯
- 关键决策与责任人可定位

------

## 6. 智能体成员需求定义

### 6.1 角色定位

智能体作为“团队第7成员”，不是问答工具，而是执行明确任务的工程角色。

### 6.2 智能体职责

1. 需求分析辅助（生成用户故事、验收标准草案）
2. 测试用例生成（正常/异常/边界）
3. 文档撰写（模块说明、接口文档、变更日志模板）
4. 部分模块代码开发（在指定目录与约束下）
5. 缺陷定位与修复建议（基于日志和测试失败信息）

### 6.3 输入输出形式

- 输入：任务单（目标、范围、约束、上下文文件）
- 输出：代码补丁建议、测试样例、文档草稿、问题定位报告
- 输出必须包含：任务ID、生成时间、模型版本、审查建议

### 6.4 人机协作要求

- AI 输出必须经人工评审后才能合并
- 每次 AI 参与任务必须绑定责任人
- 高风险代码（文件写入、批量替换）必须双人复核

------

## 7. AI 生成代码标识与管理机制（关键）

1. 分支策略
   - AI 相关变更在 `ai/*` 分支开发
2. 提交规范
   - commit message 前缀：`AI:` 或 `HUMAN:`
3. 代码标注
   - 关键片段注释：`# AI-GEN: task_id=... reviewer=...`
4. 元数据记录
   - `docs/ai-ledger.md` 记录任务、输入摘要、输出摘要、审查结论
5. PR 模板字段
   - 是否含 AI 生成内容（Y/N）
   - AI 贡献文件清单
   - 人工审核人及审核结果
6. 自动化检查
   - CI 检查无标识 AI 提交时阻断合并（可选）
