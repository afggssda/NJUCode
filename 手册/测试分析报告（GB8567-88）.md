### 3.3 重点功能测试发现

#### 3.3.1 `/doctor` 项目诊断

测试覆盖：

- doctor 检查项列表。
- selected 参数。

结论：doctor 诊断功能可以作为项目演示和验收前的健康检查工具。

#### 3.3.2 Patch 系统

测试覆盖：

- patch operation diff 生成。
- Python 语法校验。
- patch task 序列化。
- patch history 持久化。
- 修改文件的应用和回滚。
- 新建文件的应用和回滚。
- 陈旧内容拒绝。
- 路径穿越拒绝。

结果：7 个用例中 6 个通过，1 个跳过。跳过原因为当前测试沙箱禁止删除文件，影响新建文件回滚场景，不代表业务逻辑失败。

结论：patch 核心安全和修改回滚能力通过测试；在允许删除文件的普通环境中应重新执行跳过用例。

#### 3.3.3 大模型/API 客户端

测试覆盖：

- 未配置 API Key 时返回系统提示。
- 构造模型消息时注入文件上下文。
- 配置的 model_file 注入为 system 消息，缺失文件不破坏原始消息。
- 流式响应 chunk 聚合、response close 和 stop_event 中断。
- 非流式 chat 的空响应提示和 stream 异常转可读错误。

结果：相关 5 个用例全部通过。

结论：当前测试验证 API 客户端的本地逻辑，但不进行真实联网请求。该设计保证测试稳定和可复现。
